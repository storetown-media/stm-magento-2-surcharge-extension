## Das Problem

> „Bis zu **55 % der Kaufabbrüche** entstehen durch unerwartete Zusatzkosten im Checkout."

Wenn Ihr Shop Rohre, Profile, Stangenware oder sperrige Produkte verkauft, fallen häufig Zuschnitt- oder Sperrgut-Zuschläge an. Ohne transparente Darstellung dieser Kosten verlieren Sie täglich Kunden — und kassieren negative Bewertungen.

## Die Lösung

**STM Surcharge Max** zeigt Zusatzkosten **bereits auf der Produktseite** an, berechnet kategoriebasierte Zuschläge automatisch und stellt Mischkörbe transparent dar. Ab Version 2.4 mit vollwertiger **Multi-Rule-Engine** — beliebig viele benannte Regeln, Bulk-Assign, Per-Produkt-Override, alles ohne Code.

## FAQ

**Wie werden Zusatzkosten bei Mischbestellungen berechnet?**
> Bei Mischbestellungen (Kategorie-Produkte + Standardware) werden die Kosten automatisch kombiniert: Zuschnittkosten für die betroffenen Produkte plus reguläre Versandkosten für Standardprodukte.

**Ist die Extension kompatibel mit Hyvä Theme?**
> Ja — STM Surcharge Max ist vollständig kompatibel mit Hyvä Theme und Luma-basierten Themes wie Porto und Ultimo.

**Kann ich den Zusatzkosten-Hinweis individuell gestalten?**
> Ja, vollständig. Farben, Schriftgrößen, Abstände, das Info-Icon und den Hinweistext inklusive Preis-Platzhalter über die Magento System Configuration anpassen — ohne Code-Änderungen.

**Funktioniert kostenloser Versand ab einem Schwellenwert?**
> Ja. Frei wählbarer Schwellenwert (z.B. ab 100 €), ab dem der Versand für Standardprodukte automatisch kostenlos wird.

**Kann ich mehrere Zuschlags-Regeln parallel verwenden?**
> Ja. Ab Version 2.4 verwalten Sie unter **Catalog → STM Surcharge Rules** beliebig viele benannte Regeln (z.B. Cutting-Premium, Edge Polish, Engraving).

## Installation

> Composer-Installation funktioniert nach Kauf über den ausgelieferten Auth-Token / das ZIP-Paket. Eine öffentliche Packagist-Distribution ist für Q3/2026 geplant.

```bash
# Nach Kauf:
composer require storetown/module-surcharge-max
bin/magento module:enable Storetown_SurchargeMax
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento setup:static-content:deploy
bin/magento cache:flush
```

Alle Einstellungen anschließend unter **Stores → Configuration → STM → Surcharge Max** sowie unter **Catalog → STM Surcharge Rules**.
